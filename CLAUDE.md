# Project notes for Claude

Browser-only viewer for USDA's Cropland Data Layer (CDL) over Microsoft
Planetary Computer's STAC + Azure-blob COGs. No server, no derived data.
DevSeed's `deck.gl-raster` / `deck.gl-geotiff` / `geotiff` packages do the
heavy lifting. See `README.md` for the user-facing overview; this file is
for future Claude sessions.

## Where things live

All app code is under `web/`. The two Python scripts under `web/scripts/`
use PEP-723 inline deps so `uv run` handles them ‚Äî no venv, no pyproject.

```
web/src/
  App.tsx           map, MosaicLayer, dashboard, picking, filter UI
  cdlShaders.ts     three shader modules + colormap/filter texture builders
  proj.ts           EPSG:5070 <-> lon/lat (proj4 + @developmentseed/proj)
  pick.ts           out-of-band click -> CDL class code + crop name
  stats.ts          per-tile histograms + viewport-strict aggregation
  categories.ts     CDL code -> USDA category grouping
  minimal_stac.json GITIGNORED, generated by gen_stac.py (SAS-signed, ~1h TTL)
  palette.json      generated by gen_palette.py (refresh only on GEE updates)
```

## Render pipeline (integer-aware, ported May 2026)

Ported from `examples/land-cover/` in `developmentseed/deck.gl-raster`.
Per visible tile, `renderTile` wires three small shader modules in order:

1. `CreateTextureUint` ‚Äî `uniform highp usampler2D textureName`; samples
   the `r8uint` tile into `ivec4 icolor`. No normalization games.
2. `FilterCategory` ‚Äî `texelFetch(categoryFilterLUT, ivec2(icolor.r, 0), 0)`;
   `discard` if `< 0.5`. The LUT is a 256-byte boolean: `255` at every
   active class code, `0` elsewhere.
3. `PaletteColormap` ‚Äî `color = texelFetch(colormapTexture, ivec2(icolor.r, 0), 0)`;
   `discard` if `color.a == 0.0` (covers CDL class 0 "Background").

`texelFetch` everywhere means the sampler filter is irrelevant to
correctness ‚Äî no risk of bilinear blending producing class 137.5 across a
field boundary.

Two GPU textures, separate lifecycles:

- **Colormap** (256√ó1 `rgba8unorm`): built once from `palette.json`. Never
  re-uploaded.
- **Filter LUT** (256√ó1 `r8unorm`): rebuilt on every `activeCodes` change.
  256 bytes, fast.

Tile texture is `r8uint`, nearest-filter, clamp-to-edge. `r8uint` *requires*
nearest filtering (WebGL2 doesn't allow linear on integer formats).

## Picking (out-of-band)

Deck.gl's picking gives you rendered RGB, not the class code. So we don't
use it. On `onClick`:

1. Linear scan over ~1095 STAC items to find the source COG covering the
   click point (cheap; n is small).
2. Reuse `getCachedGeoTIFF(href)` ‚Äî usually a cache hit since the source is
   on screen.
3. `proj4(lng, lat)` -> EPSG:5070 meters.
4. `geotiff.index(x, y)` -> full-res `(row, col)`.
5. Compute `(tx, ty, localRow, localCol)` within the source's tile grid.
6. `geotiff.fetchTile(tx, ty)` ‚Äî usually cache-hit.
7. Read one byte -> class code -> `palette.names[code]` -> popup.

## Stats (per-tile, viewport-strict-ish)

`getTileData` calls `recordTile(image, sourceHref, x, y, data)` for each
decoded tile. That stores a 256-bucket histogram keyed by
`(sourceHref, overviewLevel, tileX, tileY)` along with the tile's lon/lat
bbox (4 corners projected from EPSG:5070).

`aggregateInViewport(bbox)` sums histograms across tiles whose bbox
intersects the viewport, weighted by overlap fraction
(see `stats.ts` for the math). Pixel area comes from each tile's affine
transform: `|a| * |e| * 0.000247105` m¬≤-to-acres. Holds at any overview
level because pixel area scales with overview.

When a source switches overview level (zoom changes), entries for the
prior level are dropped to avoid double-counting.

## SAS tokens (dev story)

`web/scripts/gen_stac.py` pre-bakes SAS-signed hrefs into
`web/src/minimal_stac.json`. Tokens expire ~1 hour after the script runs.
Symptom: 403 / "Server failed to authenticate the request" in the
network tab.

Dev-only convenience: `web/vite-plugin-regen-stac.ts` exposes a
`/regen-stac` endpoint. The app fetches it on mount; if the JSON is older
than ~50 min, the plugin re-runs `gen_stac.py` and vite's HMR picks up
the fresh file. The endpoint 404s in prod builds.

If you ever need to regen manually:

```bash
cd web
uv run scripts/gen_stac.py             # 2021 CONUS default
uv run scripts/gen_stac.py --year 2020 --bbox -125 24 -66 50
```

## Things to know before changing the render path

- **Don't use linear filtering on class textures.** Classes aren't a
  continuum. The integer pipeline already prevents this at the format
  level (`r8uint` disallows linear), but if you ever swap formats, make
  sure both the tile and palette stay on `nearest`.
- **The MosaicLayer/COGLayer wrapper.** This repo wraps `COGLayer` in
  `MosaicLayer` to fan out across ~1095 sources. Most upstream DevSeed
  examples use `COGLayer` directly against one COG. The render pipeline
  is per-tile, so the wrapper is transparent to shader-module changes ‚Äî
  but if you're porting an upstream example, expect the surrounding setup
  (`getSource`, `renderSource`, `maxCacheSize`) to look different.
- **MPC's `usda-cdl` collection caps at 2021** despite USDA publishing
  through 2024. The MPC mirror is stale. Not our problem to fix.

## Things deliberately NOT done

- GitHub Pages deployment (blocked by 1-hour SAS TTL; needs a signing
  helper with rate limits, or a cron ‚Äî neither is clean).
- Per-pixel viewport clipping in stats. Currently per-tile.
- Custom basemap. CartoCDN Voyager is fine.
